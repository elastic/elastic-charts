# Spec 18 — API documentation story (auto-extracted)

**Goal:** a Storybook page that documents the public `Trace` API as a machine-generated table
(name, type, default, required, TSDoc description), extracted from the TypeScript types via
api-extractor. The table is never hand-maintained — it is regenerated by a script and committed;
CI diffs the output.

**Depends on:** [Specs 9–15](./spec-9-color-by.md) — must be last so the generated table captures
every prop added by the earlier specs.

## Files

- `packages/charts/api-extractor.jsonc` — set `docModel.enabled: true` and
  `apiJsonFilePath: "<projectFolder>/.temp/charts.api.json"` (reuses the existing `.temp/` gitignore
  entry; the default `temp/` path would require a new ignore rule).
- `scripts/generate_trace_api_story.ts` *(new)* — reads `.temp/charts.api.json`, filters to `Trace*`
  members tagged `@public`, emits `storybook/stories/trace/19_api_docs.story.tsx`.
- `storybook/stories/trace/19_api_docs.story.tsx` *(committed, generated)* — an `EuiBasicTable`
  with columns: name, type, default, required, description. Short prose intro.
- CI wiring (e.g. `.buildkite/` or the existing `api:check` script) — run `yarn api:extract` →
  `ts-node scripts/generate_trace_api_story.ts` → `git diff --exit-code
  storybook/stories/trace/19_api_docs.story.tsx`.

## Contract

**Pipeline:**
```
yarn build:ts
yarn api:extract          # writes .temp/charts.api.json
ts-node scripts/generate_trace_api_story.ts
# 19_api_docs.story.tsx is now up to date
git diff --exit-code storybook/stories/trace/19_api_docs.story.tsx
```

The intermediate `.temp/charts.api.json` is never committed (covered by `.temp/` in `.gitignore`).

**Generated story shape:**
```tsx
// THIS FILE IS AUTO-GENERATED. Run `yarn generate:trace-api-story` to update.
import { EuiBasicTable } from '@elastic/eui';
// ... items array + columns definition
export const TraceApiDocs = () => <EuiBasicTable columns={columns} items={items} />;
```

Each row: `{ name: string; type: string; default: string; required: boolean; description: string }`.
Extract from api-extractor's `ApiInterface` members for `TraceSpec` (and `TraceStyle` for the theme
subset). TSDoc `@remarks` → `description`; `@defaultValue` → `default`; member optionality → `required`.

**Spike first (step 0):** before committing the api-extractor change, run `yarn api:extract` with
`docModel.enabled: true` and check the output:
- If `yarn api:extract` exits cleanly (warnings only about unrelated packages are non-blocking):
  proceed as planned.
- If it surfaces pre-existing TSDoc errors in the broader `@elastic/charts` surface: do **not** fix
  the whole package. Scope the generator to the `Trace*` section of the JSON and record the decision
  in this spec.

## Steps

0. **Spike:** enable `docModel.enabled: true` in `api-extractor.jsonc` locally and run
   `yarn api:extract`. Record whether it's clean or has pre-existing warnings.
1. Commit `api-extractor.jsonc` change with `apiJsonFilePath` pointing at `.temp/`.
2. Write `scripts/generate_trace_api_story.ts`:
   - Read `.temp/charts.api.json`.
   - Walk `ApiPackage → ApiInterface` entries named `TraceSpec` (and optionally `TraceStyle`).
   - For each `@public` member, extract name, type string, `@defaultValue`, optionality, `@remarks`.
   - Emit the story file with the extracted rows.
3. Run the generator; commit the initial `19_api_docs.story.tsx`.
4. Add a `generate:trace-api-story` script to `packages/charts/package.json`.
5. Wire CI to regenerate and diff.

## Storybook

`19_api_docs.story.tsx` (committed, generated):
- Short prose intro: "The `Trace` component accepts the following props via the `Trace` spec."
- `EuiBasicTable` with sortable columns: **Name**, **Type**, **Default**, **Required**,
  **Description**.
- A note at the top: "This table is auto-generated — run `yarn generate:trace-api-story` to update."

## Tests

- Unit test for `generate_trace_api_story.ts`: given a minimal fixture `charts.api.json` containing
  one `TraceSpec` interface with two members, the generator emits the expected story rows (snapshot
  test or string assertion).
- Verify the generated story compiles without TypeScript errors (`yarn typecheck`).

## Review (`/review-claudio`)

- Verify `api:extract` does not slow the normal `build:ts` or `typecheck` CI steps (it runs only in
  the `api:check` step, not in the hot build path).
- Verify the generated story is idempotent — running the generator twice produces the same output
  (no timestamps or non-deterministic ordering in the emitted code).
- Verify TSDoc descriptions are not truncated (long descriptions may span multiple TSDoc lines; the
  extractor should join them).
- Verify the spike outcome (clean vs warnings) is recorded in this spec so future maintainers
  understand the scope decision.

## Acceptance

- `yarn generate:trace-api-story` produces a `19_api_docs.story.tsx` that:
  - Renders a complete, type-derived API table for `TraceSpec`.
  - Compiles without errors.
  - Is idempotent (running twice produces no diff).
- CI regenerates and diffs the story; a prop added to `TraceSpec` without regenerating causes a
  non-zero diff → CI fails.
- `yarn api:extract` is green (or warns only on pre-existing unrelated issues, per spike outcome).
- `yarn jest trace_chart` and `yarn typecheck` are green.
